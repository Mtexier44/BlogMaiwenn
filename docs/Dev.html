💻 Projet JS : Chez Tonton Sergio

Temps de réalisation : 20h

Site vitrine non marchand – Réservation de restaurant (pizzeria)

Travail en groupe de 3 personnes

"Chez Tonton Sergio" est le tout premier projet de groupe réalisé dans le cadre de ma formation en développement web. L’objectif était de créer un site vitrine non marchand pour une activité fictive, en y intégrant les bases de la gestion de bases de données : stockage d’informations, affichage dynamique, authentification, etc.

Nous avons choisi le thème d’un restaurant italien, plus précisément une pizzeria. Le projet s’articulait autour de deux bases de données distinctes :

  • Une pour la gestion des réservations.
  • Une autre pour la gestion des utilisateurs (création de compte, profil, etc.).

Technologies utilisées :

  • HTML, CSS, JavaScript
  • Manipulation d’un SGBD
  • Travail en équipe et organisation du code
  • Gestion du temps et des tâches

Fonctionnalités développées :

  • 📅 Formulaire de réservation avec planning interactif
  • ⏰ Sélection dynamique des créneaux par date et heure
  • 🔒 Espace membre pour l’équipe du restaurant :
    • Accès aux données clients
    • Réservations
    • Partenaires, fournisseurs
    • Comptabilité simplifiée
  • 👤 Système d’authentification :
    • Création de compte
    • Connexion sécurisée

Axes d’amélioration identifiés :

  • 📁 Améliorer l’arborescence du projet
  • ❌ Rendre le système de réservation fonctionnel
  • 🎨 Revoir le design et le CSS pour plus d’harmonie
  • 🌐 Héberger le site via un véritable serveur
  • 🗃️ Remplacer la base de données en JavaScript par une solution SQL ou PHP plus adaptée

Bilan
Ce projet a été une première approche concrète du développement web. Il nous a permis d’appréhender la logique de travail en équipe, la création de fonctionnalités dynamiques et la gestion de données. Même si de nombreuses améliorations restent à faire, il a jeté des bases solides pour mes projets suivants.

🎓 Projet de Forum Web – Licence Informatique 2023-2024

Contexte du projet

Dans le cadre du projet de fin d’année du bachelor informatique, nous devions réaliser un projet réunissant l’ensemble des compétences acquises durant la première année. L’objectif est de concevoir et développer un forum web complet, en appliquant les bonnes pratiques en matière de développement logiciel, de base de données, de sécurité et de gestion de projet.

Présentation du projet

Le forum doit permettre :

  • La communication entre utilisateurs (messages et commentaires)
  • L’association de catégories aux messages
  • Un système de likes / dislikes
  • Le filtrage des messages

Base de données – SQLite

Les données seront stockées dans une base SQLite. Le projet doit contenir au minimum une requête SELECT, CREATE et INSERT.

Il est également demandé de concevoir un modèle entité-association.

Authentification

  • Inscription avec e-mail, nom d’utilisateur et mot de passe
  • Rejet si l’e-mail est déjà utilisé
  • Le mot de passe doit être chiffré (bonus)
  • Utilisation de cookies pour gérer une session unique par utilisateur
  • Expiration de session définie par les étudiants
  • L’utilisation d’un UUID pour la session est un bonus

Communication

  • Seuls les utilisateurs enregistrés peuvent créer des messages/commentaires
  • Les messages peuvent être associés à des catégories
  • Les visiteurs non connectés peuvent uniquement consulter le contenu

Likes / Dislikes

  • Fonction réservée aux utilisateurs connectés
  • Le nombre de likes/dislikes est visible par tous

Filtrage

Les messages doivent pouvoir être filtrés par :

  • Catégories
  • Messages créés par l’utilisateur connecté
  • Messages likés par l’utilisateur connecté

Les deux derniers filtres sont réservés aux utilisateurs connectés.

Contraintes techniques

  • Utilisation obligatoire de SQLite
  • Gestion des erreurs et des statuts HTTP
  • Bonne structuration du code
  • Tests unitaires recommandés

Packages autorisés

  • Packages standards de Go
  • sqlite3
  • bcrypt
  • uuid

Interdiction d’utiliser des frameworks frontend (React, Vue, etc.)

Objectifs pédagogiques

Ce projet permettra de renforcer les compétences suivantes :

  • Notions web : HTML, HTTP, sessions, cookies
  • Langage SQL
  • Manipulation de base de données
  • Notions de chiffrement
